# データ準備ガイド

Aggyで集計を行うために必要な **ローデータCSV** と **レイアウトJSON** の作成方法を説明します。

## ファイルの概要

集計には以下の2ファイルが必要です。

| ファイル | 役割 |
|---------|------|
| ローデータCSV | アンケートの回答データ本体 |
| レイアウトJSON | 各CSV列の種別（SA/MA/NA/WEIGHT/DATE/MATRIX）と選択肢ラベルの定義 |

### ローデータCSV

- 文字コード: **UTF-8**
- 1行目: ヘッダー（列名）
- 2行目以降: データ行

### レイアウトJSON

CSVの各列がどの種類の設問かを定義するJSON配列です。

```json
[
  { "key": "列名", "type": "種別", ... },
  ...
]
```

## 列タイプ別の定義

### WEIGHT — ウェイト列

ウェイトバック集計で使用する重み列です。ウェイト不要なら省略できます。

| | 内容 |
|--|------|
| CSV列名 | `key` と一致（例: `weight`） |
| CSVの値 | 小数（`1.2`, `0.9` 等） |
| レイアウト | `{ "key": "weight", "type": "WEIGHT" }` |

`items` は不要です。

### SA — 単一回答

1列に選択肢コード（数値）が入ります。

| | 内容 |
|--|------|
| CSV列名 | `key` と一致（例: `q1`） |
| CSVの値 | 選択肢コード（`1`, `2`, `99` 等） |

```json
{
  "key": "q1",
  "label": "性別",
  "type": "SA",
  "items": [
    { "code": "1", "label": "男性" },
    { "code": "2", "label": "女性" },
    { "code": "3", "label": "その他" },
    { "code": "99", "label": "無回答" }
  ]
}
```

- `key`: CSV列名と一致させる
- `label`: 集計結果に表示される設問名
- `items`: 選択肢の一覧。`code` はCSVに入っている値、`label` は表示名

### MA — 複数回答

選択肢ごとに1列ずつCSV列を持ちます。列名は **`{key}_{code}`** の形式です。

| | 内容 |
|--|------|
| CSV列名 | `{key}_{code}`（例: `q3_1`, `q3_2`, `q3_3`） |
| CSVの値 | 選択した=`1`、選択していない=`0` |

```json
{
  "key": "q3",
  "label": "利用しているサービス",
  "type": "MA",
  "items": [
    { "code": "1", "label": "サービスA" },
    { "code": "2", "label": "サービスB" },
    { "code": "3", "label": "サービスC" }
  ]
}
```

- `key`: MA列グループの接頭辞（CSV列名の `_` より前の部分）
- `items`: 各選択肢。`code` はCSV列名のサフィックス（`q3_1` の `1`）

### DATE — 日付列

日付データを指定した粒度（年・月・週・日）で集計します。`items` は不要で、データから自動生成されます。

| | 内容 |
|--|------|
| CSV列名 | `key` と一致（例: `answer_date`） |
| CSVの値 | `YYYY-MM-DD` 形式の日付文字列（例: `2025-01-15`） |

```json
{
  "key": "answer_date",
  "label": "回答日",
  "type": "DATE",
  "granularity": "month"
}
```

- `key`: CSV列名と一致させる
- `label`: 集計結果に表示される設問名
- `granularity`: 集計粒度。以下のいずれか:
  - `"year"` — 年単位（例: `2025`）
  - `"month"` — 月単位（例: `2025-01`）
  - `"week"` — ISO 8601 週単位（例: `2025-W09`）
  - `"day"` — 日単位（例: `2025-01-15`）
- `items` は指定不要（データから自動生成される）
- NULL値は集計から除外されます

### MATRIX — マトリクス設問

同じ選択肢を共有する複数の子設問をグループ化して、1つのテーブルにまとめて表示します。「サービスA/B/Cそれぞれの満足度」のように、同じ軸で複数の対象を評価する設問に使います。

マトリクスは **MATRIX親エントリ** と **子設問エントリ** の組み合わせで定義します。

#### MATRIX親エントリ

グループの見出しを定義します。CSV列は持ちません。

```json
{
  "key": "q4",
  "label": "サービス別満足度",
  "type": "MATRIX"
}
```

- `key`: 子設問の `matrixKey` から参照される識別子
- `label`: 集計結果に表示されるグループ名

#### 子設問エントリ

通常のSA/MA/NA設問に `matrixKey` を追加します。CSV列の持ち方は各型と同じです。

```json
{
  "key": "q4_1",
  "label": "サービスA",
  "type": "SA",
  "matrixKey": "q4",
  "items": [
    { "code": "1", "label": "満足" },
    { "code": "2", "label": "普通" },
    { "code": "3", "label": "不満" }
  ]
}
```

- `matrixKey`: MATRIX親の `key` と一致させる
- それ以外のフィールドは通常のSA/MA/NA定義と同じ

#### NA型のマトリクス

数値設問もマトリクスにできます。GT表示では子設問を行、統計量（n/mean/median等）を列にした1テーブルになります。

```json
{ "key": "q5", "label": "サービス別評価スコア", "type": "MATRIX" },
{ "key": "q5_1", "label": "サービスA", "type": "NA", "matrixKey": "q5" },
{ "key": "q5_2", "label": "サービスB", "type": "NA", "matrixKey": "q5" },
{ "key": "q5_3", "label": "サービスC", "type": "NA", "matrixKey": "q5" }
```

#### 注意事項

- MATRIX親はCSV列を持たないため、CSVにはMATRIX親の `key` に対応する列は不要です
- `matrixKey` に指定した値に対応する `type: "MATRIX"` のエントリが同じレイアウト内に存在しないとエラーになります
- 子設問がCSVに1つも存在しない場合、MATRIX親も自動的に除去されます

## CSVの例

以下のCSVとレイアウトが対になります。

```csv
id,weight,q1,q2,q3_1,q3_2,q3_3
1,1.2,1,3,1,0,1
2,0.9,2,1,0,1,1
3,1.5,1,2,1,1,0
11,1.0,99,99,0,0,0
12,0.8,1,,0,0,0
```

| レイアウト定義 | CSV列名 |
|--------------|---------|
| `"key": "weight", "type": "WEIGHT"` | `weight` |
| `"key": "q1", "type": "SA"` | `q1` |
| `"key": "q2", "type": "SA"` | `q2` |
| `"key": "q3", "type": "MA"` (items: 1,2,3) | `q3_1`, `q3_2`, `q3_3` |

## 無回答と回答対象外

「無回答」と「回答対象外」は意味が異なり、集計での扱いも違います。

- **無回答**: 設問が表示されたが、回答者が回答しなかった
- **回答対象外**: そもそも設問が表示されなかった（分岐条件で非該当など）

### SA の場合

| 状態 | CSVの値 | 集計での扱い |
|------|---------|-------------|
| 通常の回答 | 選択肢コード（`1`, `2` 等） | 該当コードにカウント |
| 無回答 | 選択肢コード（例: `99`） | 「無回答」としてカウント（nに含まれる） |
| 回答対象外 | 空セル（NULL） | 集計から除外（nに含まれない） |

無回答を集計結果に含めるには、`items` に無回答用の選択肢を追加し、CSVにもそのコードを入れます。

```json
{ "code": "99", "label": "無回答" }
```

### MA の場合

| 状態 | CSVの値 | 集計での扱い |
|------|---------|-------------|
| 選択した | `1` | 該当項目にカウント |
| 選択していない | `0` | カウントされない |
| 無回答 | 全列が `0` | nに含まれるが、どの項目にもカウントされない |
| 回答対象外 | 全列が空セル（NULL） | 集計から除外（nに含まれない） |

## 注意事項

- **集計に不要な列はCSV・レイアウトの両方に含めない** でください。レイアウトに定義されていないCSV列は無視されますが、不要な列を含めるとデータが煩雑になります
- `code` は文字列として定義してください（`"1"` であって `1` ではない）
- JSON Schemaによるバリデーションが可能です: `public/schemas/layout.schema.json`
